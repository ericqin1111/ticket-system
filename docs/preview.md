购票通
本项目基于SpringCloud和Springboot，构建了一套高并发的微服务分布式购票系统。架构上拆分为库存、订单、票务、用户及网关五大核心服务，旨在解决高流量下的“超卖”与“高可用”难题

高并发库存与一致性： 解决了“超卖”问题。采用 Redis + Lua 处理高并发库存预扣减，并结合 Debezium + Kafka 构建事务消息（Outbox）机制，实现核心业务异步解耦与最终一致性
订单超时自动关闭： 引入 RocketMQ 延时队列，实现了订单超时自动关闭及库存回填，解决明明有货但买不到的错误体验
统一认证与鉴权： 设计并实现了JWT 无状态认证方案。在网关层统一鉴权，实现用户身份的解析与下游服务透传，简化微服务间调用链路
高性能多级缓存： 构建 Caffeine + Redis 的多级缓存架构，采用旁路缓存模式提升热点数据响应速度，并设计了缓存一致性保障方案
分布式流量削峰：实现了基于 Redis 排队 ZSET 的分布式流量整形（遵循漏桶思想）。采用 Lua 脚本保证匀速放行的原子性，并通过策略模式实现了按用户 ID/业务 ID 等多种粒度的可扩展放行，完美适配秒杀削峰需求
分布式链路追踪： 集成 PLG（Grafana + Loki） 搭建日志监控平台，通过 TraceID 贯穿业务流程，实现跨服务链路追踪，提升线上问题排查效率
高可用保障： 针对分布式环境，设计并实现了核心接口的幂等性与消息消费幂等性，保障系统在重试和网络分区下的数据准确性
